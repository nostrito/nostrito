Documentation

Everything you need to set up and configure your personal Nostr relay. nostrito is currently in active development — docs will expand as features ship.

Getting Started

nostrito is a personal Nostr relay that runs on your machine. It syncs your feed, caches media, and serves everything locally over WebSocket.

Installation

Download the latest release for your platform from the GitHub releases page, or build from source:

git clone https://github.com/nostrito/nostrito.git cd nostrito cargo build --release

Quick Start

Open the app and paste your npub. nostrito handles the rest — database setup, relay connections, and trust graph computation.

Configuration

nostrito uses a TOML configuration file stored alongside its database. Most settings can be configured through the GUI, but power users can edit the file directly.

Config file location

  • ~/.config/nostrito/config.toml on Linux
  • ~/Library/Application Support/nostrito/config.toml on macOS
  • %APPDATA%\nostrito\config.toml on Windows

Relay Aliases

nostrito lets you refer to relays by name instead of WebSocket URLs. Built-in aliases are provided for popular relays.

Built-in aliases

  • primal — Fast global relay
  • damus — iOS community hub

  • snort — Web client relay
  • coracle — Discovery-focused

Storage

nostrito gives you fine-grained control over what gets stored locally. Your own events are always kept. Others' events can be filtered by kind, trust depth, and quota.

Storage tiers

  • Your events — Always stored, 100%, no exceptions
  • Others' events — Configurable by WoT depth and storage quota
  • Media (Blossom) — Images, videos, and audio cached locally with hash verification

Web of Trust

nostrito computes your trust graph from your follow list, recursively. It uses this to decide whose events to sync and store.

How it works

Starting from your npub, nostrito fetches your follow list (kind 3), then your follows' follow lists, and so on up to a configurable depth. Each hop reduces trust weight.

  • 1 hop — People you directly follow
  • 2 hops — People your follows follow
  • 3 hops — Extended network (lower trust weight)

Advanced

Advanced configuration options for power users.

Relay server

nostrito runs a NIP-01 compliant WebSocket relay on localhost:4869 by default. You can change the port and bind address in settings.

Daemon mode

nostrito can run as a background daemon that starts with your system and syncs continuously.

FAQ

What NIPs does nostrito support?

nostrito implements NIP-01 (basic protocol) with plans to support additional NIPs as needed. The goal is compatibility with all major Nostr clients.

Does nostrito work offline?

Yes. Once events and media are synced, everything is available locally. Your relay serves cached data even without an internet connection.

How much disk space does nostrito use?

It depends on your configuration. Your own events typically use under 500 MB. Others' events and media are configurable with storage quotas.

Can I use nostrito with any Nostr client?

Any client that supports adding a custom relay can connect to nostrito. Just add your local relay address in your client's relay settings.
